## 学习记录

### 模块 A — AI 基础：核心概念

**LLM 本质**：基于上下文进行概率生成，预测最合理的下一个 token 序列。

**四个关键概念**：
- **Token**：最小读取单位，≈0.75 英文词 / 1 中文字
- **Embedding**：文字转数学向量，表达语义相似度
- **Transformer**：注意力机制让模型关注输入中重要的部分
- **Hallucination（幻觉）**：自信编造不存在的信息——必须外部核实

**Prompt → Workflow → Agent 三层次**：

| 层次 | 决策者 | 风险 |
|------|--------|------|
| Prompt | 人在决策 | 低 |
| Workflow | 预定义流程 | 中 |
| Agent | 模型自主规划+工具调用 | 高 |

**Agent 核心组件**：Tool Calling、MCP、Skills、Tracing、Guardrails、Handoff、错误恢复、长期记忆。

### 模块 A — Prompt 深度

- Instruction 四段式：角色→边界→异常处理→输出格式
- Few-shot：放示例让模型模仿判断
- Structured Output：JSON/schema 返回，用于后续检查
- Prompt Injection：输入净化 + 权限分离 + 输出验证

### 模块 A — Agent 深度

- 核心分工：模型提方案→系统限空间→用户批边界
- 三大支柱：权限分级、可观测性、停止条件
- Tool Use：schema、只读/写入、副作用、人工确认
- Planning：拆步骤，暴露工具/读写/确认/重试
- 失败模式：循环、工具滥用、幻觉传递、权限蔓延
