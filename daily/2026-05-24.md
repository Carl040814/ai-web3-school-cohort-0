## 学习记录

### Bridge 1 — 链感知上下文

核心：模型不能凭记忆判断链上事实——余额、授权、合约地址都在实时变化。Agent 上下文必须带 chain id / block number / citation。

七个知识节点：On-chain Data、Contract Docs、ABI/Event、Transaction History、Explorer Context、Indexing Context、Citation。

一个完整链感知上下文包：用户目标 + chain id + 地址余额 + 合约 ABI + 最近交易 + 索引更新时间 + citation。

> 知识库：`bridge/chain-aware-context.md`

### Bridge 2 — Web3 工具调用

核心：把 RPC、合约读写、钱包确认、DeFi 操作变成 Agent 可调用工具。关键不是"能调用"，而是权限、参数、模拟和日志。

权限分层：查询余额自动 → 交易草稿自动 → 小额 session key → 大额转账必须人工确认 → 任意合约调用默认禁止。

八个知识节点：RPC Tool、Contract Read/Write、Wallet Tool、Explorer Tool、DeFi Tool、Tool Permission、Tool Log。

> 知识库：`bridge/web3-tool-use.md`

### Bridge 3 — Agent 工作流

核心：把概率模型放进确定性流程——Task Graph、State Machine、Human-in-the-loop、Retry/Fallback、Trace、Evaluation Harness、Regression Set。

一个完整 swap 工作流：读取目标→获取余额→查价格→生成交易→模拟→展示风险→用户确认→发送→追踪。每步有输入/输出/权限/停止条件。

> 知识库：`bridge/agent-workflow.md`

### Bridge 4 — Agent 钱包

核心：控制权不能交给一个概率系统。Agent 只能拿到可验证、可限制、可撤销的行动空间。

九大组件：AA Wallet、Smart Account、Safe、Session Key、Policy、Guard、Simulation、Revocation、Human Check。

完整链路：用户目标→Agent 计划→受限交易→Guard+Simulation→用户确认→Smart Account 执行→日志。

> 知识库：`bridge/agent-wallet.md`
